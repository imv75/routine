## New investigation this session: Playwright / headless Chromium

This environment now ships a pre-installed Chromium + Playwright-compatible setup
(`/opt/pw-browsers`), which prior sessions did not have confirmed working (their
`scripts/scrape_usc_playwright.py` and `scripts/test_playwright_acalog.py` were committed but never
produced output data). Since several of the 26 remaining blockers are JS-rendered SPAs or Acalog WAF
challenges that a plain `requests`-based scraper cannot pass, a real browser seemed like the most
promising untried avenue.

**Result: not viable.** Installed Playwright (`npm install playwright`) and drove the bundled Chromium
(141.0.7390.37) directly. Every HTTPS navigation through the session's TLS-intercepting egress proxy
fails with `net::ERR_CONNECTION_CLOSED` (`SSL error code 1`) — including trivial control cases like
`https://example.com`, which rules out site-specific blocking. Root cause (via `--log-net-log`):
Chromium's BoringSSL includes a post-quantum hybrid key share (X25519MLKEM768) in its TLS 1.3
ClientHello by default; the proxy's TLS-interception layer cannot handle it, and closes the connection
mid-handshake. Plain `curl`/Python `requests` (OpenSSL-based, no PQ key share) go through the identical
proxy without issue, confirming the break is specific to Chromium's TLS stack, not the proxy's general
policy or the destination sites.

Attempted fixes, all unsuccessful:
- `--disable-quic`, `--disable-http2`
- `--disable-features=PostQuantumKyber,UseMLKEM,X25519MLKEM768,TLS13KyberSupport,EncryptedClientHello,X25519Kyber768`
- Chrome enterprise managed policy `PostQuantumKeyAgreementEnabled: false` (via
  `/etc/chromium/policies/managed/policy.json`)
- Persistent profile + NSS trust store (already provisioned per `/root/.ccr/README.md`)

This is an environment/infrastructure limitation (Chromium TLS vs. this specific proxy), not something
fixable from within the scraper code. It explains why the earlier `scrape_usc_playwright.py` /
`test_playwright_acalog.py` scripts from a prior session never produced any data.
