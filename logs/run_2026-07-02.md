## New investigation this session: Internet Archive Wayback Machine

Prior sessions tried direct `requests`-based scraping (blocked) and headless Chromium via Playwright
(blocked by a Chromium-specific TLS/proxy incompatibility). Neither had tried retrieving *archived*
snapshots of the blocked catalogs via `web.archive.org`, which is reachable from this proxy (unlike the
live origin domains for many of these schools) and doesn't require a live browser or a live connection to
the target university's servers.

**Findings:**
- `web.archive.org` and its CDX API are reachable and return real results for most of the 26 targets
  (e.g. `umich`, `utk`), which is a genuinely new capability not available to prior sessions.
- For Acalog-based catalogs (utk, and by extension the other Acalog/CourseLeaf-style `dynamic_js_only`
  schools), the archived subject-listing pages (`content.php?catoid=X&navoid=Y`) only contain course
  *titles* — full descriptions are loaded client-side via AJAX from a separate `preview_course_nopop.php`
  page per course, and those per-course pages were, in the crawls checked, not consistently archived
  alongside the listing page.
- Acalog's server-rendered "print" view (`...&print`, which normally embeds full descriptions inline
  without AJAX and would avoid needing one archived request per course) does have CDX entries, but
  fetching the exact archived snapshot returned Wayback's own 404 despite the CDX record showing
  `statuscode: 200` for that exact URL+timestamp — likely a query-string canonicalization mismatch
  between what CDX indexed and what Wayback's replay resolver expects. Resolving this reliably would
  need per-CMS reverse-engineering of Wayback's URL matching, then a scraper that fetches on the order of
  thousands of individual archived course pages per university-year (department listing → per-course
  page → parse), across ~10-25 years, per school.
- That volume of fetches (likely tens of thousands of Wayback requests per school to reach the same
  depth of coverage the 74 completed schools have) is not something to rush through in a single pass —
  doing it half-complete (e.g. one year, a handful of departments) risks landing inconsistent,
  not-actually-comparable data in the dataset next to the 74 schools that do have full 2000-present
  coverage.

**Conclusion:** the Wayback Machine is a real, previously-untried avenue and is worth a dedicated future
session to build a proper per-CMS archived-page scraper (starting with Acalog, which covers ~9 of the 15
`dynamic_js_only` schools). It is not a same-session quick win. No new universities marked completed today.
